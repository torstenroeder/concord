Person Concordancer 1.2

This is a tool to find general identifiers for persons, just by providing some basic data. This is version 1.2.

Method

Person data is compared in sets of (predefined) contexts. Basically, every information needs to be confirmed by a second information in the same context to be plausible. As contexts can overlap, it is possible that one information can be evaluated more than once.

Every person that matches a context gains points according to the relevance of the context. Points are accumulated from all context comparisons. The highest possible score can differ, depending on the request.

Parameters

You can submit a request by a combination of URL parameters.

  • n - principal name (can contain more than one word)
  • on - other names (can contain more than one word)
  • db - date of birth (ISO 8601)
  • pb - place of birth
  • dd - date of death (ISO 8601)
  • pd - place of death
  • d - description keywords

Hints

  • names/descriptions can consist of more than one word

Examples

  • a request with many details
  • a request with only name and description
  • a request without a name

Results

  • results will be returned in XML
  • if there are more than results, the following results will be displayed with a score of or more
  • more than results will not be displayed
